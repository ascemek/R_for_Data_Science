Workflow and tips on data wrangling

R for Data Science

Some notes on dplyr

The dplyr package is one of the packages contained in the tidyverse collection. It specializes in helping us wrangle data sets select subsets, arrange data, create new variables, etc.

As you may have noticed in the reading, every dplyr function follows a similar format:

  • The first argument is the name of the data frame
  • The subsequent arguments (separated by commas) tell R what to do to that data. Because weve already told R what data set we are working with, we can refer to variable names directly (without the $ we learned about in week 1)
  • Each function returns a data frame. That means that if you just type the function into your console, the result will try to print the ENTIRE resulting data frame. You may want to consider saving your results to a new data frame instead (e.g.new_data <- select(old_data, var1, var2, var3)). More on that next

Workflow with dplyr

When working with data wrangling and transformation, there are two general purposes:

  • Change our data set for use later. We may want to create subsets (filter) or new variables (mutate) to use in later visualizations or analyses. In this case, its very important to make sure we save the results of our data wrangling to keep for later use.
  • Display summaries or unusual cases. We may just want to highlight the mean and standard deviation in each sub-group. Or we may want to print out the top five values. In this case, our purpose is to inform our audience, but we dont necessarily need to save the results of our work for later. Printing the output will suffice.

These two purposes lead to three possible workflows with data wrangling.

Change data for later use

Often when we create new variables or subsets, we are doing so with the intent of graphing or modeling that data later. In this case, its important to save the results of our work.

Whats wrong with the following code? Why does it result in an error?

StudentSurvey %>%
  mutate(BMI = Weight / Height^2 * 703)

ggplot(StudentSurvey, aes(x = Exercise, y = BMI)) + 
  geom_point()

# Error in FUN(X[[i]], ...) : object 'BMI' not found

Display summaries or unusual cases

The other purpose of data wrangling is to display information directly, rather than using it later. This happens often when creating summary tables with group_by() and summarise(). These are usually intended to be printed as they are, and you may not need to save them first.

StudentSurvey %>%
  group_by(Year) %>%
  summarize(meanGPA = mean(GPA, na.rm = TRUE), 
            NumStudents = n(), 
            NumMissing = sum(is.na(GPA)))
## `summarise()` ungrouping output (override with `.groups` argument)

## 1 ""           NaN              2          2
## 2 "FirstYear"    3.08          94         13
## 3 "Junior"       3.24          35          0
## 4 "Senior"       3.17          36          0
## 5 "Sophomore"    3.18         195          2

No need to save this!

We may also only want to print the results of arrange() to highlight the highest or lowest values. For example, what were the five highest SAT scores among students in this sample?

StudentSurvey %>%
  select(Year, SAT, GPA) %>%
  arrange(desc(SAT)) %>%
  head(5) # prints the first five rows

So to summarize:

# Save the results if you need this later
# Usually do this with mutate, filter, etc.
new_data <- old_data %>%
  mutate(new_var = old_var * 2)

# Print group summaries with group_by() and summarise() or 
# Highlight cases with arrange() and head()
# May occasionally need to save, but often just print
old_data %>%
  group_by(var1) %>%
  summarize(meanvar2 = mean(var2))

Some trouble-shooting tips:

  • These functions assume you are working with a tidy data frame. That is, each column represents one variable, and each row represents one case.
  • You may run into issues because other packages have similarly named functions (there are several functions called select(), for example). If R gives you trouble about recognizing the function name, you can explicitly refer to the function within dplyr using ::. For example, dplyr::select(my_data, var1) tells R to use the select() function in dplyr.

More on grouped summaries

Most useful:

  • group_by + summarize
  • grouping by multiple variables
  • common summary functions (mean, sd, etc.)
  • count functions (n(), count())
  • counting missing and non-missing data with sum(is.na()) and sum(!is.na())
  • finding the proportion of missing data with mean(is.na())

Instead of wanting one summary for an entire variable, we want to compare distributions among different levels of another category. Weve seen how to do this visually (e.g.by faceting our graph), and now we will learn how to do the same for our data and numerical summaries.

my_data %>%
  group_by(categorical_variable) %>%
  summarize(mean_value = mean(quant_variable, na.rm = TRUE))

What is the average GPA of students in this survey? Now that weve checked how many missing values we have, lets tell R its ok to ignore the missing data:

mean(StudentSurvey$GPA, na.rm = TRUE)
## [1] 3.157942
StudentSurvey %>%
  summarize(mean_GPA = mean(GPA, na.rm = TRUE))
##   mean_GPA
## 1 3.157942

Well, thats ok to start with, but its not a terribly interesting question to answer. We could keep going just with GPA find other summaries, or graph the distribution of GPA but lets try to ask a more complicated question instead.

What is the average GPA among each year (first year, sophomore, etc.)? Does one year tend to have higher GPAs?

To do this, we need to group the data by year first, group all the first years together and find their average GPA; then group the second years together and find their GPA, and so on.

We could do this by filtering:

StudentSurvey %>%
  filter(Year == "FirstYear") %>%
  summarize(mean_gpa = mean(GPA, na.rm = TRUE))
##   mean_gpa
## 1 3.077654

But that would get old very quickly, since wed have to create separate code for each year. Instead, lets tell R to group the data:

StudentSurvey %>%
  group_by(Year) %>%
  summarize(mean_gpa = mean(GPA, na.rm = TRUE))
## `summarise()` ungrouping output (override with `.groups` argument)
## # A tibble: 5 x 2
##   Year        mean_gpa
##   <fct>          <dbl>
## 1 ""            NaN   
## 2 "FirstYear"     3.08
## 3 "Junior"        3.24
## 4 "Senior"        3.17
## 5 "Sophomore"     3.18

While were at it, lets get R to find the number missing in each group, as well as the number who actually responded.

StudentSurvey %>%
  group_by(Year) %>%
  summarize(mean_gpa = mean(GPA, na.rm = TRUE), 
            num_missing = sum(is.na(GPA)), 
            num_respond = sum(!is.na(GPA)),

There is a lot more we could do. For instance, we could find the quartiles (five number summary) for our data.

quantile(StudentSurvey$GPA, na.rm = TRUE)
##   0%  25%  50%  75% 100% 
##  2.0  2.9  3.2  3.4  4.0

However, this ends up being a bit tricky to put inside a call to summarize():

StudentSurvey %>%
  summarize(quart_gpa = quantile(StudentSurvey$GPA, na.rm = TRUE))
# Error: Column `quart_gpa` must be length 1 (a summary value), not 5

What went wrong? summarize() can only enter one value for each summary. But here, the quantile() function is trying to shove an entire five number summary into a place where theres only room for one value.

So what if we want the five number summary? We need to split it up by value:

StudentSurvey %>%
  summarize(gpa_min = min(GPA, na.rm = TRUE), 
            gpa_Q1 = quantile(GPA, 0.25, na.rm = TRUE), 
            gpa_Q2 = quantile(GPA, 0.5, na.rm = TRUE),
            gpa_Q3 = quantile(GPA, 0.75, na.rm = TRUE),
            gpa_max = max(GPA, na.rm = TRUE))
##   gpa_min gpa_Q1 gpa_Q2 gpa_Q3 gpa_max
## 1       2    2.9    3.2    3.4       4

That took a fair bit of work, and we could have just accomplished the same thing from the built in quantile() or fivenum() functions without using summarize! But the nice thing with the tidyverse approach is that we can easily extend it. Want the five number summary of GPA for each year? No problem:

StudentSurvey %>%

Rename a variable

Sometimes a variable name will be unclear, and you may want to change it. This is easy to do with the rename() function:

rename(my_data, new_name = old_name)

The one catch is that the above code doesnt save the results! As we saw earlier, we need to save our updated data in a new data frame (or write over our existing data frame, but thats usually a bad idea unless we are certain we know what we are doing).

new_df <- my_data %>%
  rename(new_name = old_name)

Recode a categorical variable

Notice that gender in our data set is recorded as F or M:

levels(StudentSurvey$Gender)
## [1] "F" "M"

While there are obvious issues here about recording gender as a binary variable in the first place, we might at least want to choose clearer labels such as Female or Male. We can do this by recoding:

student_new <- StudentSurvey %>%
  mutate(Gender = recode(Gender, 
         "F" = "Female", 
         "M" = "Male"))

Convert a numerical variable to categorical

Consider the distribution of exercise times for students:

fivenum(StudentSurvey$Exercise)
## [1]  0  5  8 12 40
ggplot(StudentSurvey, aes(y = Exercise)) + 
  geom_boxplot() + 
  coord_flip()
## Warning: Removed 1 rows containing non-finite values (stat_boxplot).

There are several outliers here, which could make later analyses tricky. What if we try grouping each person into a category? Something like Little, Some, Moderate, High, or Extremely high amounts of exercise?

We can do this, using the case_when() function. This lets us create new variables (with mutate()), when certain criteria (cases) occur. For example:

StudentSurvey <- StudentSurvey %>%
  mutate(Exercise_Cat = case_when(
    Exercise >= 20 ~ "Extremely high", 
    Exercise >= 12 ~ "High", 
    Exercise >= 8 ~ "Moderate", 
    Exercise >= 4 ~ "Some", 
    Exercise < 4 ~ "Little"))

ggplot(StudentSurvey, aes(x = Exercise_Cat, y = Exercise)) + 
  geom_boxplot()
## Warning: Removed 1 rows containing non-finite values (stat_boxplot).

That created a new variable called Exercise_Cat, with each student categorized by their exercise amount. However, our graph is confusing R orders the categories alphabetically! Wed like to tell R the correct order for our categories.

Change order of categories for categorical variable

We can use the function fct_relevel() to change the order of the categories for a categorical variable. This is considered data wrangling rather than data visualization, because it actually changes how R views our data set itself. After we change the order, the updated order will appear on ALL graphs, tables, models, etc.

# Reorder categories
ex_levels <- c("Little", "Some", "Moderate", "High", "Extremely high")

StudentSurvey <- StudentSurvey %>%
  mutate(Exercise_Cat = fct_relevel(Exercise_Cat, ex_levels))

ggplot(StudentSurvey, aes(x = Exercise_Cat, y = Exercise)) + 
  geom_boxplot()
## Warning: Removed 1 rows containing non-finite values (stat_boxplot).

Much better!

From here, we can group by Exercise category and summarize other variables within each exercise category. Note again that R preserves the new order of the categories on all future tables, graphs, etc.:

StudentSurvey %>%
  group_by(Exercise_Cat) %>%
  summarize(SAT_min = min(SAT, na.rm = TRUE), 
            SAT_mean = mean(SAT, na.rm = TRUE), 
            SAT_max = max(SAT, na.rm = TRUE), 
            Num_students = n())
